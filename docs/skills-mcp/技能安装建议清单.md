# Skills 使用与维护清单（仅保留已安装）

## 1. 当前状态

- 已安装 skills：20 个（来源：`skills list -g --json`）
- 本文档只维护“已安装能力”，未安装/替代项全部移除
- 维护原则：先用现有能力覆盖需求，再决定是否新增安装

## 2. 已安装 Skills：功能与触发条件

### 2.1 基础与通用

- `find-skills`
  - 功能：检索并发现可安装 skills，给出安装建议。
  - 触发条件：用户问“有没有 skill 能做 X”“帮我找个 skill”。
  - 安装指令：`npx --registry=https://registry.npmjs.org -y skills add vercel-labs/skills@find-skills -g -y`
  - GitHub 地址：`https://github.com/vercel-labs/skills`

- `technical-writer`
  - 功能：编写技术文档、说明文档、操作指引。
  - 触发条件：用户要写 README、方案文档、教程、知识库文章。
  - 安装指令：`npx --registry=https://registry.npmjs.org -y skills add obra/superpowers@technical-writer -g -y`
  - GitHub 地址：`https://github.com/obra/superpowers`

- `api-documentation-writer`
  - 功能：输出 API 文档（接口说明、示例、错误码、鉴权等）。
  - 触发条件：用户要整理接口文档、补齐 OpenAPI/Swagger 信息。
  - 安装指令：`npx --registry=https://registry.npmjs.org -y skills add obra/superpowers@api-documentation-writer -g -y`
  - GitHub 地址：`https://github.com/obra/superpowers`

## 3. 可剔除项提示（仅提示，不能卸载）

> 说明：以下仅用于“低频提醒”，不执行卸载操作。

- 可优先观察是否低频：`vercel-react-native-skills`
  - 原因：当前项目主线是 Web React，若无 RN 需求可长期低频。

- 可优先观察是否低频：`api-endpoint-scaffolder`
  - 原因：若项目几乎不做后端接口开发，该能力调用频率可能偏低。

- 可优先观察是否低频：`audit-website`
  - 原因：偏周期性体检，不是每次开发都用到。

## 4. 维护约定

- 每月复核一次“最近 30 天使用频率”。
- 连续两个月低频的 skill，标记为“可剔除候选”。
- 若出现明确新场景，再恢复候选 skill 的优先级。
